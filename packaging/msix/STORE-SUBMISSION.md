# tracker keeper — Microsoft Store (MSIX) submission

GENERATED by `trackerkeeper bake`. The **first** submission is manual via Partner Center
(the `runFullTrust` capability triggers a 1–5 day cert review); after that,
`.github/workflows/msix.yml` can automate updates via the `msstore` CLI.

## One-time: register the identity

1. Create a **free individual** developer account at
   <https://partner.microsoft.com/dashboard> and reserve the app name **tracker keeper**.
2. From the app's **Product identity** page, copy:
   - **Package/Identity Name** → must match `Name="wolfgangwarehaus.trackerkeeper"` in `AppxManifest.xml`.
   - **Publisher (CN=…)** → fill `store_secrets_of_record.msix_publisher_cn` in
     `pyproject.toml`, then re-run `trackerkeeper bake` (the manifest `Publisher` is blank until then).
   - **Package Family Name** → `msix_pfn`; **Store ID** → `msix_store_id` (tracked, never regenerated).

## Build the .msix locally (or run msix.yml once activated)

```sh
# 1. Freeze the Windows onedir (from a Windows checkout):
pyinstaller packaging/pyinstaller/trackerkeeper.spec --noconfirm
# 2. Render the tile/logo assets from the brand SVG:
bash packaging/msix/make-assets.sh
# 3. Stage manifest + assets next to the frozen bundle and pack (Windows SDK):
copy packaging\msix\AppxManifest.xml dist\trackerkeeper\
xcopy /E /I packaging\msix\Assets dist\trackerkeeper\Assets
makepri createconfig /cf priconfig.xml /dq en-US /o
makepri new /pr dist\trackerkeeper /cf priconfig.xml /of dist\trackerkeeper\resources.pri /o
makeappx pack /d dist\trackerkeeper /p dist\trackerkeeper.msix /o
```

> The manifest's `Publisher="CN="` is empty until you fill `msix_publisher_cn` and
> re-bake — `makeappx` **fails** on an empty/mismatched publisher, so do step 2 first.

The Store **re-signs** at ingestion, so the Store-bound `.msix` needs no
code-signing cert — that re-sign is the whole point of this channel (it fixes
SmartScreen for free). Pre-declare the PyInstaller-bootloader false positive in
the submission notes.

## Validate locally before submitting

A failed `runFullTrust` review costs 1–5 days, so prove the package installs +
passes certification on a **test-signed** copy first (full-trust Win32 — the
frameless chrome, blur, single-instance, and the `windows.startupTask` autostart —
only behaves correctly when actually installed under `WindowsApps`, not from `dist`).

```powershell
# A self-signed cert whose Subject == a test Publisher; rebuild a -test copy with
# that Publisher, sideload it, then run the Windows App Certification Kit.
$cert = New-SelfSignedCertificate -Type Custom -Subject "CN=trackerkeeper-test" `
  -KeyUsage DigitalSignature -CertStoreLocation "Cert:\CurrentUser\My" `
  -TextExtension @("2.5.29.37={text}1.3.6.1.5.5.7.3.3")
# ... export it, rewrite Publisher to CN=trackerkeeper-test in a dist\trackerkeeper-test
# copy, makeappx pack + signtool sign that copy with the cert, then:
Add-AppxPackage dist\trackerkeeper-test.msix
# QA the INSTALLED app: launch, toggle "Launch at login" (StartupTask must arm in
# Settings > Apps > Startup), confirm single-instance + the frameless chrome.
& "${env:ProgramFiles(x86)}\Windows Kits\10\App Certification Kit\appcert.exe" `
  test -appxpackagepath dist\trackerkeeper.msix -reportoutputpath wack.xml
```

## Submit

Upload `dist\trackerkeeper.msix` in Partner Center → **Packages**, justify
`runFullTrust` (a packaged classic desktop app using Win32 APIs), and submit.
Free products' updates are automatable thereafter via `msix.yml` + the Store secrets.

## Version

The MSIX `Version` is a 4-part Store version **decoupled** from the marketing
version (first part must be `!= 0`, fourth `= 0`). Bump it per submission — it is
NOT the git tag / marketing version (a `0.x.y` tag isn't even a valid first part).
